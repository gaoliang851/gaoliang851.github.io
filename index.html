Swift5.1学习笔记之下标

发表于 2020-09-03 分类于 iOS ， Swift

定义和使用

下标(subscript)可以给任意类型(枚举，结构体，类)增加下标功能，也叫做**下标脚本**

  • subscript的语法类似于实例方法、计算属性，本质上就是方法(函数)

  • subscript中可以定义set方法和get方法，接受的参数即为下标传入的值，set方法中使用newValue来接受设置的值

  • subscript中定义的返回值类型决定了：

    • get方法的返回值类型
    • set方法中newValue的类型

  • subscript可以接受多个参数，并且类型任意(eg:String)

下标的使用细节

  • subscript可以没有set方法，但必须要有get方法

  • 如果只有get方法，可以省略get

  • 可以设置参数标签，外部调用使用参数标签，内部使用形参

  • 下标可以是类型方法，并且要使用static修饰

Swift5.1学习笔记之继承

发表于 2020-09-03 分类于 iOS ， Swift
  • 值类型(枚举、结构体)不支持继承，只有类支持继承
  • 没有父类的类，称为：基类
    • Swift并没有像OC，Java那样的规定：任何类最终都要继承自某个基类
  • 子类可以重写父类的下标、方法、属性，重写必须加上override关键字

内存结构

Swift中，实例的内存布局：前八个字节存储class的类型信息，第二个八个字节存放着该实例的引用计数信息。后面的内存存放着实例的属性数据。在Swift中，为实例分配的字节数必须是16的倍数。

例如：在如下场景中，有基类Animal,Dog继承自Animal，ErHa继承自Dog。

Animal的实例在内存中，前16个字节，分别存放着类型信息和引用计数信息，第三个八个字节存放着属性age的数据，攻击占24个字节。因为内存分配必须是16的倍数，所以Animal的一个实例会占有32个字节。

Dog继承自Animal,前16个字节，依旧分别存放着类型信息和引用计数信息。第三个八个字节存放着从父类继承来的属性age的属性，第四个八个字节存放着自身weight属性的数据。一般来说，在内存中，从父类继承来的属性会排列在自身属性前边。

ErHa继承自Dog,前16个字节，依旧分别存放着类型信息和引用计数信息，第三个和第四个八个字节分别存放着属性age和weight的属性，第五个八个字节存放着iq的数据，一个实例占用40个字节，因为内存分配必须是16的倍数，所以ErHa的一个实例会占有48个字节。

重写属性

  • 子类可以将父类的属性(存储、计算)重写为计算属性
  • 子类不可以将父类属性重写为存储属性
  • 只能重写**var属性，不能重写let**属性
  • 重写时，属性名、类型要一致
  • 子类重写后属性权限不能小于父类属性的权限
    • 如果父类属性是只读的，那么子类重写后属性可以是只读的，也可以是可读写的。
    • 如果父类属性是可读写的，那么子类重写后的属性也是必须可读写的。

实例属性

  • 子类将父类的存储属性重写为计算属性时，储存属性的内存依旧会分配，如果想要访问这段存储要使用super。

      如果配置的路径中，包含不存在的目录会报错，如果没有数据库文件则会自动创建

    • 数据库设置只读属性，如果对设置readOnly的数据库，提交写入事务会崩溃抛出异常

      假如获取 Bundle中的数据库：

      [[NSBundle mainBundle] URLForResource:@"222" withExtension:@"realm"]

      程序包的数据文件虽然是只读的，但没有设置readOnly属性，向数据库中添加数据，也可以添加成功并查询出结果。但是程序重启后，添加的数据就消失了。

      猜测是：

      1. 将Bundle中的数据库拷贝出来一份临时数据库文件，重启时删除这个临时文件
      2. 将Bundle中的数据库加载到了内存中，后续的数据修改并没有落盘。

主键

重写 +[RLMObject primaryKey] 可以设置模型的主键。声明主键允许对象的查询和更新更加高效，并且会强制要求每个值保持唯一性。一旦将带有主键的对象添加到 Realm 数据库，那么该对象的主键将无法更改。

索引属性

要为某个属性建立索引，那么重写 +[RLMObject indexedProperties] 即可。与主键类似，索引会稍微减慢写入速度，但是使用比较运算符进行查询的速度将会更快（它同样会造成 Realm 文件体积的增大，因为需要存储索引。）当您需要为某些特定情况优化读取性能的时候，那么最好添加索引。

Realm 支持为字符串、整型、布尔值以及 NSDate 属性建立索引。

数据库迁移

对于多个版本程序，数据库模型发生了变化，一定要注意对数据库进行迁移

Realm中主要通过RLMRealmConfiguration中的schemaVersion以及migrationBlock属性来对老版本数据进行迁移。迁移的工作 +[AppDelegate didFinishLaunchingWithOptions:] 中进行配置。

迁移的场景中包含：数据结构的迁移，数据的迁移，属性重命名等。

数据结构的迁移

  • 对于某个属性是否是必须属性的变化

    Realm可以自己处理，不需要去干预。如果某个属性从非必须属性变为必须属性,对于之前的该属性为nil的数据，会自动添加上默认值，比如String从nil变为""

  • 对于新增或删除属性的变化

    Realm 会自行检测新增和需要移除的属性，然后自动更新硬盘上的数据库架构

数据迁移

以Person对象为例：在v1版本中，包含三个属性age、firstName和lastName。在v2版中变为两个属性age和fullName,其中fullName是firstName+lastName拼接而来。

属性重命名

两种方式：

  • 新属性的值等于旧属性的值

  • 调用migration的重命名方法

多版本跳跃性迁移

对于多个版本跳跃性的迁移，要从`

  • RLMRealmConfiguration *config = [RLMRealmConfiguration defaultConfiguration];
// 设置新的架构版本。这个版本号必须高于之前所用的版本号
uint64_t newVersion = 1;
config.schemaVersion = newVersion;
config.migrationBlock = ^(RLMMigration * _Nonnull migration, uint64_t oldSchemaVersion){
  //迁移要做的工作
  if (newVersion > oldSchemaVersion) {            
    [migration renamePropertyForClass:Person.className oldName:@"oldProperty" newName:@"newProperty"];
  }
};
